Artifacts via tool: model compiles/validates server-side, reply uses [[artifact:N]]

New create_artifact tool: the model submits a mermaid/chart/html spec, the server
COMPILES/validates it (mermaid via jsdom+mermaid.parse in an isolated worker
thread; chart via JSON; html non-empty) and returns any error so the model fixes
and retries until valid. On success it gets index N and writes [[artifact:N]] in
the reply, so the message stays clean and only validated artifacts render.

- New deps: mermaid + jsdom (server-side validation only; client still CDN-loads).
- mermaid-worker.mjs runs jsdom/mermaid in a worker thread so browser globals
  do not leak into the Next server; validate.ts wraps it (fails open if down).
- artifacts persisted on messages (new column), streamed via MEDIA_MARKER, parsed
  client-side, rendered through the existing Artifact component at [[artifact:N]].
- Prompt now directs the model to use create_artifact instead of raw code blocks.
